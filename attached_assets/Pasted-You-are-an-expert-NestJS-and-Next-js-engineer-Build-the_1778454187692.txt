PART 1 — NestJS API: EventEnquiriesModule
═══════════════════════════════════════════════════════

Create apps/api/src/modules/event-enquiries/ with:

1. event-enquiries.controller.ts — 5 endpoints:

GET    /v1/event-enquiries
  - Customers see own enquiries
  - Vendors see enquiries where they submitted a quote (or were matched)
  - Admin sees all
  - Filter by status: open, quoted, confirmed, completed, cancelled

POST   /v1/event-enquiries (customer role)
  - Validates: eventDate must be at least 7 days from now (events need planning time)
  - guestCount minimum 10
  - Triggers vendor matching immediately (see Step 3)
  - Returns created enquiry

GET    /v1/event-enquiries/:id
  - Returns enquiry with all submitted quotes (customer and matched vendors only)

POST   /v1/event-enquiries/:id/select-vendor (customer role)
  Body: { vendorId: string }
  - Validates the vendor submitted a quote for this enquiry
  - Updates enquiry: vendorId, status=confirmed
  - Calculates deposit (30% of selectedQuote.perHeadPence * guestCount + deliveryFeePence)
  - Creates Stripe PaymentIntent for deposit amount with capture_method: 'manual'
  - Sets other quotes to status=expired
  - Returns { enquiry, clientSecret } for deposit payment

PATCH  /v1/event-enquiries/:id/confirm-numbers (customer role)
  Body: { guestCount: number, menuAdjustments?: string }
  - Updates final guest count
  - Recalculates final total
  - Creates balance PaymentIntent (70% of final total) — collected 48h before event
  - Returns updated enquiry

POST   /v1/event-enquiries/:id/quotes (vendor role)
  Body: { proposedMenu, perHeadPence, deliveryFeePence, minDepositPct, terms, expiresAt }
  - Validates vendor was matched to this enquiry
  - Validates quote deadline not passed (24h from enquiry creation)
  - Status=submitted
  - Notifies customer: "New quote received for your event"

2. event-enquiries.service.ts:

   matchVendors(enquiry: EventEnquiry):
   - Find vendors where status=live
   - Filter by: vendor's delivery radius covers enquiry.deliveryPostcode
     (use postcodes.io to get lat/lng for both, then haversine distance)
   - Filter by: at least one of vendor.cuisineTypes matches enquiry.cuisinePreferences
     (if cuisinePreferences is empty, match all vendors)
   - Limit to 5 vendors maximum
   - Send each matched vendor a notification: new event enquiry
   - Store matched vendor IDs (add matched_vendor_ids varchar[] to EventEnquiry model)

   scheduleBalanceCollection(enquiryId: string):
   - Add BullMQ delayed job: fires 48h before eventDate
   - Job creates Stripe PaymentIntent for balance and sends customer payment link

3. Add to prisma/schema.prisma:
   In EventEnquiry model, add: matchedVendorIds String[] @map("matched_vendor_ids") @db.Uuid
   Then run: npx prisma migrate dev --name add_event_matched_vendors

PART 2 — BullMQ cron jobs for events
═══════════════════════════════════════════════════════

5. In the compliance/scheduler module, add two new cron jobs:

   event-reminder (runs every hour):
   - Find EventEnquiry where status=confirmed AND eventDate = now() + 72 hours ± 1 hour
   - For each: send customer reminder to confirm final guest numbers
   - Send email and WhatsApp notification

   event-balance (runs every hour):
   - Find EventEnquiry where status=confirmed AND eventDate = now() + 48 hours ± 1 hour
     AND balancePiId IS NULL
   - For each: create Stripe PaymentIntent for balance amount
   - Send customer payment link via email and WhatsApp
