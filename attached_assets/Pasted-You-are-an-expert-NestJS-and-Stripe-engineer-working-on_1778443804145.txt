AUDIT 1 — Stripe PaymentIntent escrow (capture_method)
═══════════════════════════════════════════════════════

1. Open apps/api/src/modules/orders/orders.service.ts
   Find the stripe.paymentIntents.create() call.
   
   It MUST include: capture_method: 'manual'
   
   If it does NOT have capture_method: 'manual':
   Add it now. The full create call should be:
   
   const paymentIntent = await this.stripe.paymentIntents.create({
     amount: totalPence,
     currency: 'gbp',
     capture_method: 'manual',
     metadata: {
       orderId: order.id,
       vendorId: dto.vendorId,
       customerId: customerId,
     },
     description: `Feastpot order from ${vendorName}`,
   });
   
   Without capture_method: 'manual', customers are charged immediately on order placement
   regardless of whether the vendor accepts. This is the Feastpot escrow mechanism.

2. Find where the order is marked as delivered (updateStatus with status=delivered).
   Confirm stripe.paymentIntents.capture(stripePaymentIntentId) is called at this point.
   
   If missing, add it:
   if (newStatus === OrderStatus.delivered) {
     await this.stripe.paymentIntents.capture(order.stripePaymentIntentId);
     // Update commissionPence and netToVendorPence in the order record
   }

3. Find where the order is rejected or cancelled.
   Confirm stripe.paymentIntents.cancel(stripePaymentIntentId) is called.
   If missing, add it:
   if (newStatus === OrderStatus.rejected || newStatus === OrderStatus.cancelled) {
     await this.stripe.paymentIntents.cancel(order.stripePaymentIntentId);
   }

AUDIT 2 — 15-minute auto-cancel BullMQ job
═══════════════════════════════════════════════════════

4. Find the confirmOrder() method in orders.service.ts.
   
   It MUST enqueue a delayed BullMQ job after the vendor is notified:

     'auto_cancel',

       delay: 15 * 60 * 1000, // 15 minutes
       jobId: `auto-cancel-${order.id}`, // deterministic jobId for cancellation
     }
   );
   
   If this job is NOT present, add it now.

5. Find the updateStatus() method where vendor accepts (status = accepted).
   It MUST remove the auto-cancel job:
   
   if (newStatus === OrderStatus.accepted) {
     const job = await this.ordersQueue.getJob(`auto-cancel-${orderId}`);
     if (job) await job.remove();
   }
   
   If missing, add it.

6. Find the orders.processor.ts (or equivalent BullMQ processor).
   It MUST handle the 'auto_cancel' job type:
   
   case 'auto_cancel': {
     const order = await this.ordersRepository.findById(data.orderId);
     if (order && order.status === OrderStatus.pending) {
       await this.stripe.paymentIntents.cancel(order.stripePaymentIntentId);
       await this.ordersRepository.updateStatus(data.orderId, OrderStatus.cancelled, 'auto_rejected_vendor_timeout');
       // Enqueue customer notification: order auto-cancelled
       await this.notificationsQueue.add('send_email', {
         to: order.customer.email,
         subject: 'Your Feastpot order was not accepted',
         template: 'order_auto_cancelled',
         data: { vendorName: order.vendor.businessName },
       });
     }
     break;
   }
