STEP 3 — Update apps/api/src/main.ts for production:
- Listen on 0.0.0.0 (not localhost) — required for Replit container networking
- PORT from process.env.PORT || 3001
- Trust proxy (for Replit's reverse proxy): app.set('trust proxy', 1)
- Add startup health check log: console.log('Feastpot API running on port', port)

STEP 4 — Configure Sentry in apps/api:
Install: @sentry/nestjs @sentry/node @sentry/profiling-node

Create apps/api/src/instrument.ts (must be imported first in main.ts):
import * as Sentry from '@sentry/nestjs'
Sentry.init({
  dsn: process.env.SENTRY_DSN,
  environment: process.env.NODE_ENV,
  integrations: [Sentry.nestjsIntegration()],
  tracesSampleRate: process.env.NODE_ENV === 'production' ? 0.1 : 1.0,
  profilesSampleRate: 0.1,
})

Configure SentryGlobalFilter in app.module.ts
Configure source maps in nest-cli.json for readable stack traces in Sentry

STEP 6 — Deployment steps in Replit:
1. Click Deploy button in Replit workspace
2. Select Autoscale deployment type
3. Machine: 1 vCPU, 2GB RAM, min instances: 1, max: 10
4. Health check path: /health
5. Click "Link a domain" → enter api.feastpot.co.uk
6. Copy the A record and TXT record Replit provides
7. Add both to Cloudflare DNS (DNS Only, no proxy)
8. Wait for Replit to verify the domain (60-120 seconds)
9. Click Deploy

STEP 7 — Post-deployment verification:
Run these from your local machine or Replit shell:

curl -X POST https://api.feastpot.co.uk/v1/auth/login -d '{}' (should return 400 validation error, not 500)
